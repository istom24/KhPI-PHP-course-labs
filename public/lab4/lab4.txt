Лабораторна робота № 4
Тема: Об'єктно-орієнтоване програмування в PHP.
Мета: Ознайомитися з основами об'єктно-орієнтованого
програмування в PHP. Навчитися створювати класи,
використовувати специфікатори доступу, конструктори та
реалізовувати спадкування.
Завдання:
Реалізувати систему класів для роботи з простим
інтернет-магазином, який містить інформацію про товари та
категорії товарів.
Крок 1: Створення базових класів
1. Створіть клас Product, який буде представляти товар.
Клас повинен мати такі властивості:
● name (Назва товару)
● price (Ціна товару)
● description (Опис товару)
2. Використовуйте специфікатори доступу для властивостей:
● name та description повинні мати публічний
доступ.
● price має бути захищеним (protected), щоб до
нього можна було отримати доступ через дочірні
класи, але не напряму.
3. Створіть конструктор класу Product, який ініціалізує
властивості товару.
4. Реалізуйте метод getInfo(), який виводить інформацію
про товар у форматі:
Назва: Назва товару
Ціна: Ціна товару
Опис: Опис товару
Крок 2: Спадкування та розширення функціоналу
1. Створіть клас DiscountedProduct, який успадковує клас
Product.
● Цей клас повинен мати додаткову властивість
discount (знижка на товар).
● Створіть конструктор, який ініціалізує всі властивості
з урахуванням батьківського класу.
● Додайте метод для розрахунку нової ціни з
урахуванням знижки.
● Перевизначте метод getInfo() для того, щоб
виводити інформацію про знижку та нову ціну.
Крок 3: Створення та тестування об'єктів
1. Створіть кілька об'єктів класу Product та
DiscountedProduct.
2. Виведіть інформацію про кожен товар за допомогою методу
getInfo().
Завдання:
1. Реалізуйте класи Product та DiscountedProduct
відповідно до описаних вимог.
2. Створіть кілька товарів та виведіть їх інформацію на
сторінці.
3. Реалізуйте додатковий клас Category, який
представлятиме категорії товарів:
● Властивості: назва категорії, масив товарів.
● Метод для додавання товару до категорії.
● Метод для виведення всіх товарів категорії.
● Додайте валідацію ціни товару в класі Product, щоб
вона не могла бути від’ємною.
Звіт:
Звіт має включати в себе довільний набір php файлів.
